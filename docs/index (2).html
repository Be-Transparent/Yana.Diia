Yana.Diia.AI

LLM-Driven Requirements Engineering для державних сервісів

Provider-agnostic AI платформа для інтеграції з Diia. Специфічно спроєктована для вирішення реальних проблем: автоматизація BRD-генерації, Diia Design System compliance, LLM-as-a-Judge оцінка якості.

Проблеми, які ми вирішуємо

1. Ручна генерація BRD — затратно й неконсистентно

Дизайнери та PM витрачають 20+ годин на створення UI flows, user stories й Figma макетів. Результати часто не узгоджені з Diia Design System.

Рішення: LLM-as-Generator автоматично створює BRD, JSON specs, Mermaid диаграми та Figma UI на основі природної мови. LLM-as-Judge перевіряє на Diia D-DS compliance.

2. Тестування якості UI flows — немає об'єктивних метрик

Як виміряти "хороший" flow? BLEU, ROUGE не враховують UX, доступність, державні вимоги.

Рішення: Diia Flow Scoring Rubric (5 метрик): Flow Length, Component Compliance, Screen Saturation, Faithfulness, Explainability. Пairwise LLM-Judge порівнює варіанти.

3. Інтеграція з 16 державних API без централізованого управління

Finance.AI потребує доступу до Diia SSO, fiscal.gov.ua, naz.gov.ua, United24 та ін. Кожна інтеграція — окремий проект.

Рішення: API Hub + RAG над Diia Design System. LLM генерує код для API calls на основі документації. CodeMie / AWS LiteLLM для безпечного виконання.

Архітектура

LLM-Driven RE Pipeline

  • Ideation Phase: Natura language → User Stories, Acceptance Criteria
  • LLM-as-Generator: BRD → JSON specs, Mermaid, Figma designs
  • LLM-as-Judge: Evaluate flow quality (5 metrics)
  • Pairwise Judge: Порівняння N варіантів, вибір оптимального
  • Golden Dataset: Pre-production evaluation benchmark
